\noindent \textbf{待排序序列是 $(33,31,13,54,82,33,40,72)$, 有 8 个处理器.}\\
因为是示例, 不妨假设每次都是从左往右第一个处理器竞争到写入, 则有如下二叉树: \\
\begin{center}
\begin{forest}
[33
	[31
		[13]
		[,phantom]
	]
	[54
		[33
			[,phantom]
			[40]
		]
		[82
			[72]
			[,phantom]
		]
	]
]
\end{forest}
\end{center}
说明: 
\begin{itemize}
\item 从 root 开始, 第一个是 33, 而后分为 [31, 13] 和 [54, 82, 33, 40, 72]
	\begin{itemize}
	\item 左边 [31, 13] 中, 31 竞争到了, 于是先设置了 31, 然后自然是 13
	\item 右边 [54, 82, 33, 40, 72] 中, 54 竞争到了, 于是先设置了 54, 分为了 [33, 40] 和 [82, 72]; 然后更进一步分别是 33 和 82
	\end{itemize}
\end{itemize}

\section*{2.}
\textbf{二叉排序树并行地转化为有序数组, 写出算法, 时间复杂度, 并行计算模型.}\\

不妨考虑类似\textbf{顺序统计树}的做法, 由此可以直接在 $O(\log n)$ 内得到每个节点的次序, 然后直接写入数组对应元素地址即可. 这里主要问题是统计每个节点对应的子树大小.

\begin{algorithm}[H]
	\caption{{\sc Sorted-Tree-To-Array}}
	\begin{algorithmic}[1] %每行显示行号
		\Function {{\sc Sorted-Tree-To-Array}}{T}
			\For {i=1 to n \textbf{par-}} \Comment{初始化 size 数组 和 rank 数组, $O(1)$}
				\State size[i] = 0
				\State rank[i] = -1
			\EndFor
			\For {i=1 to n \textbf{par-}} \Comment{初始化 parent 数组, $O(1)$}
				\If {$LC_i$ available}
					\State parent[$LC_i$] = i
				\EndIf
				\If {$RC_i$ available}
					\State parent[$RC_i$] = i
				\EndIf
			\EndFor
			\For {k = 1 to $\ceil{1+\log n}$} \Comment{初始化 level 数组, 自上而下遍历每层, $O(\log n)$}
				\For {proc i in level k \textbf{par-}}
					\State level[i] = k
				\EndFor
			\EndFor
			\For {k = $\ceil{1+\log n}$ to 1} \Comment{逐层上传统计数量, $O(\log n)$}
				\For {proc i in level k \textbf{par-}}
					\State $size[i] = size[RC_i] + size[LC_i]$
				\EndFor
			\EndFor
			\State
			\For {i=1 to n \textbf{par-}} \Comment{利用顺序统计数的方法计算每个节点的 rank, $O(\log n)$}
				\State $rank[i] = size[LC_i] + 1$
				\State $j = i$
				\While {$j != root$}
					\If {$value[j] < value[i]$}
						\State $rank[i] = rank[i] + size[LC_j] + 1$
					\EndIf
					\State $j = parent[j]$
				\EndWhile
			\EndFor
			\For {i=1 to n \textbf{par-}} \Comment{写入数组, O(1)}
				\State array[rank[i]] = value[i]
			\EndFor
			\State \Return array
		\EndFunction
	\end{algorithmic}
\end{algorithm}

根据前面的分析知道, \textbf{时间复杂度是 $O(\log n)$ 的}, 所采用的并行计算模型是\textbf{PRAM-CRCW}.
